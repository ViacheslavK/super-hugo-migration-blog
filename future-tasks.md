# Future Tasks

## og:image для постів (cover-зображення)

### Проблема
Більшість постів мають `cover = ""` у front matter. Коли посилання поширюється у соцмережах (Facebook, Telegram, LinkedIn, X), тег `og:image` підставляє `favicon.ico` замість повноцінного зображення — превью виглядає непрезентабельно.

### Рішення (рекомендоване: комбінований підхід)

**Крок 1 — Підготувати fallback-зображення**

Створити файл `static/og-default.jpg`:
- Розмір: **1200 × 627 px**
- Вміст: логотип блогу, назва "jUst QA", можливо підзаголовок
- Інструменти: Canva, Figma або будь-який графічний редактор
- Формат: JPG (оптимальний розмір файлу до 300 КБ)

**Крок 2 — Оновити `layouts/partials/head.html`**

Замінити fallback у блоці OG-даних (рядки ~68-73) з `favicon.ico` на `og-default.jpg`:

```html
{{ else }}
  <meta property="og:image" content="{{ "og-default.jpg" | absURL }}">
{{ end }}
```

**Крок 3 — Для нових постів додавати cover вручну**

У front matter нового посту:
```toml
cover = "назва-файлу.jpg"
```
Файл покласти поруч з `index.md` або у `content/uploads/YYYY/`.

### Що залишається без змін
Пости, де `cover` вже заповнений, використовують своє зображення автоматично — логіка в шаблоні вже є.

### Пріоритет
Середній — впливає на CTR у соцмережах, але не на індексацію Google.
